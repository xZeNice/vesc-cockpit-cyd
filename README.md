# VESC Cockpit для CYD (ESP32‑2432S028R)

Прошивка‑«приборка» для электротранспорта на базе контроллеров **VESC**, работающая на дешёвом дисплейном модуле **CYD (Cheap Yellow Display, ESP32‑2432S028R)**.

Читает телеметрию с VESC по UART, рисует анимированную приборную панель с тач‑интерфейсом и добавляет кучу полезного сверху: рекорды, посотовый мониторинг батареи, диагностику, разгонный таймер (Dragy), Wi‑Fi веб‑дашборд, мост для VESC Tool и BLE‑пульт блокировки.

> ⚠️ Прошивка управляет транспортным средством (в т.ч. блокировкой газа). Используйте на свой страх и риск и тестируйте на подставке/без нагрузки.

## 🔌 Подключение VESC → CYD

Связь идёт по **UART** (аппаратный Serial1 ESP32). На стороне CYD используются выведенные на разъёмы пины:

| Сигнал CYD | GPIO | Куда на VESC | Примечание |
|---|---|---|---|
| `VESC_RX` | **GPIO 22** | → **TX** порта VESC | приём данных от VESC |
| `VESC_TX` | **GPIO 27** | → **RX** порта VESC | отправка команд в VESC |
| `GND` | `GND` | → **GND** VESC | общая земля обязательна |

```
   CYD (ESP32)                         VESC (COMM / UART)
 ┌──────────────┐                    ┌──────────────────┐
 │  GPIO22 (RX) ●──────────────────► ● TX               │
 │  GPIO27 (TX) ●◄────────────────── ● RX               │
 │        GND   ●──────────────────── ● GND             │
 └──────────────┘                    └──────────────────┘
```

> 🔁 **Главное правило: RX↔TX крест‑накрест.** RX дисплея идёт на TX контроллера и наоборот.
>
> ⚡ UART у VESC — логика **3.3 В**, ESP32 тоже 3.3 В → подключаются напрямую, без конвертера уровней.
>
> 🧩 Скорость UART фиксирована на **115200 бод** (`SERIAL_8N1`). В VESC Tool включите App → General → **UART**, baud 115200.

### Тачскрин
Пины тача (`MOSI=32, MISO=39, CLK=25, CS=33`) разведены на плате CYD и **отдельно паять не нужно** — они уже подключены к XPT2046.

### Второй VESC (опционально)
Если контроллеров два и они связаны по **CAN**, второй мотор/ESC опрашивается через VESC‑CAN‑forwarding. В прошивке задаётся **CAN ID** второго устройства (экран `CAN ID`). Подключать второй VESC к дисплею не нужно — данные идут через первый по CAN.

## 🔋 Питание

CYD питается **5 В** (USB‑разъём или пин `5V/VIN`). Варианты:

- от **USB** (отладка / стенд);
- от **5 В выхода VESC** (если модель его имеет) — тогда дисплей включается вместе с транспортом;
- от отдельного **DC‑DC 5 В**, запитанного от батареи.

> Не подавайте напряжение батареи напрямую на 5 В пин — только через стабилизатор.

## 🛠 Среда сборки и библиотеки

**Arduino IDE** (или PlatformIO), плата — **ESP32 Dev Module**.

### Partition Scheme (важно!)
Прошивка использует BLE + Wi‑Fi + OTA, поэтому нужна схема разделов с местом под OTA:

```
Tools → Partition Scheme → Minimal SPIFFS (1.9MB APP with OTA / 128KB SPIFFS)
```
Если этого не выбрать — сборка остановится с ошибкой (в коде стоит проверка `#error`).

### Библиотеки

| Библиотека | Назначение |
|---|---|
| `VescUart` | обмен с VESC по UART |
| `TFT_eSPI` | вывод на TFT ILI9341 |
| `XPT2046_Touchscreen` | резистивный тач |
| `SimpleKalmanFilter` | сглаживание телеметрии |
| `ArduinoOTA`, `ESPmDNS`, `WiFiUdp` | обновление «по воздуху» |
| `WebServer`, `DNSServer` | веб‑дашборд + captive‑портал |
| `BLEDevice` / `BLE2902` | BLE‑пульт и мост VESC Tool |
| `Preferences` | сохранение настроек в NVS |

### Настройка TFT_eSPI под CYD (обязательно!)

Библиотека `TFT_eSPI` настраивается **один раз** через файл `User_Setup.h`, который лежит в папке самой библиотеки:

```
Arduino/libraries/TFT_eSPI/User_Setup.h
```

Открой его и приведи к виду для **ESP32‑2432S028R** (закомментируй чужие драйверы и впиши это):

```cpp
#define ILI9341_2_DRIVER        // драйвер дисплея CYD (если цвета инвертированы — попробуй ILI9341_DRIVER)

#define TFT_WIDTH  240
#define TFT_HEIGHT 320

// --- SPI-пины дисплея на плате CYD ---
#define TFT_MISO 12
#define TFT_MOSI 13
#define TFT_SCLK 14
#define TFT_CS   15
#define TFT_DC    2
#define TFT_RST  -1            // RST соединён с EN платы
#define TFT_BL   21            // подсветка
#define TFT_BACKLIGHT_ON HIGH

// --- шрифты ---
#define LOAD_GLCD
#define LOAD_FONT2
#define LOAD_FONT4
#define LOAD_FONT6
#define LOAD_FONT7
#define LOAD_FONT8
#define LOAD_GFXFF
#define SMOOTH_FONT

#define SPI_FREQUENCY       55000000
#define SPI_READ_FREQUENCY  20000000
```

> 📌 **Тач здесь настраивать НЕ нужно.** Резистивный XPT2046 обслуживается отдельной библиотекой `XPT2046_Touchscreen` прямо в скетче (пины `32/39/25/33`), а не через TFT_eSPI — поэтому `TOUCH_CS` в `User_Setup.h` не задаётся.
>
> 🎨 Если после прошивки цвета инвертированы или красный/синий перепутаны — поменяй драйвер на `#define ILI9341_DRIVER` (без `_2`).

**Откуда взять эталонный конфиг и схемы пинов:** официальный community‑репозиторий CYD — [github.com/witnessmenow/ESP32-Cheap-Yellow-Display](https://github.com/witnessmenow/ESP32-Cheap-Yellow-Display) (там готовый `User_Setup.h`, схемы пинов и примеры). Это самый надёжный источник правды по железу CYD.

## ⚙️ Первичная настройка

Все параметры задаются прямо на дисплее (раздел настроек) и сохраняются в энергонезависимой памяти (NVS). Значения по умолчанию:

| Параметр | По умолчанию | Диапазон | Где |
|---|---|---|---|
| Кол‑во полюсов мотора | 30 | 2–60 (чётное) | Motor config |
| Диаметр колеса, мм | 280 | 80–800 | Motor config |
| Передаточное число ×100 | 100 (=1.00) | 10–1000 | Motor config |
| Число ячеек батареи (S) | 10 | 10–24 | Settings |
| Калибровка напряжения ×1000 | 1000 (=1.00) | 800–1200 | Settings |
| CAN ID второго VESC | 1 | — / OFF | CAN ID |

> Полюса/колесо/редуктор нужны для **корректной скорости и пробега**, число ячеек — для расчёта **% заряда (SOC)**, калибровка — чтобы показания вольтажа совпадали с мультиметром.

## 🚀 Функционал прошивки

### Приборная панель (главный экран)
- Анимированная **дуга‑спидометр** с цветовой градацией скорости.
- Крупная цифра скорости (км/ч), индикация **рекуперации**.
- Колонки: **VOLT**, **WATT**, **TRIP KM**.
- Анимированные «жидкостные» колонки **заряда** и **мощности** (с эффектом волны и «молнии» при зарядке).
- Бейджи‑предупреждения: блокировка газа, перегрев мотора (≥85 °C), перегрев FET (≥80 °C), ошибки VESC.

### Телеметрия
- Сглаживание показаний фильтром **Калмана**.
- Поддержка **двух VESC** (основной по UART + второй по CAN): суммарная мощность, max‑температуры, статусы.
- Авто‑детект потери связи с VESC (таймаут 3 с).

### Рекорды и пробег
- Хранение **рекордов** (max скорость, мощность, ток, температуры FET/мотора) и **общего одометра** в NVS.
- Накопительный пробег и Wh, расчёт **эффективности (Wh/км)**.
- Сброс рекордов с подтверждением (PIN).

### Диагностика и батарея
- Экран **Cells** — напряжение по ячейкам и состояние пакета.
- Экран **Diag** — живые параметры VESC.
- Экран **CAN Devices** — обнаруженные по CAN контроллеры.

### Dragy — разгонный таймер
- Замер времени разгона (как Dragy/Race‑box) прямо на дисплее.

### Безопасность / блокировка
- **Блокировка курка газа** через VESC (программная, не зависит от BLE).
- **PIN‑код** на разблокировку, экранная цифровая клавиатура.
- **Авто‑блокировка** по таймеру простоя.
- **Ограничение скорости** (speed limit).
- Опция блокировки **при старте**.

### Прочее
- **Музыка** — проигрывание мелодий/сигналов.
- Несколько **тем оформления** (палитры).
- Экранная **QWERTY‑клавиатура** для ввода Wi‑Fi.

## 📡 Беспроводные сервисы

### 1. Веб‑дашборд (Wi‑Fi AP)
ESP32 поднимает собственную точку доступа и отдаёт веб‑страницу с телеметрией.

| Параметр | Значение |
|---|---|
| SSID | `VESC-Dash` |
| Пароль | `vesc12345` |
| Адрес | `http://192.168.4.1` |
| Captive‑портал | да (DNS на порту 53 — страница откроется сама) |

Доступны API `/api` (JSON‑телеметрия, free heap, аптайм) и `/cmd` (команды).

> 🧠 **Управление автоматическое:** веб‑дашборд **включается при входе на свой экран и выключается при выходе**. Это сделано специально — активный Wi‑Fi AP + BLE съедают RAM, из‑за чего спрайты колонок заряда/мощности на главном экране могли не отрисоваться. На время работы AP BLE‑пульт временно отключается и возвращается при выходе.

### 2. Мост для VESC Tool (BLE)
ESP32 прикидывается BLE‑адаптером VESC (Nordic UART Service), и официальное приложение **VESC Tool** может подключиться к контроллеру «через дисплей».

| Параметр | Значение |
|---|---|
| Имя устройства | `VESC-Tool` |
| Сервис (NUS) | `6e400001-b5a3-f393-e0a9-e50e24dcca9e` |

### 3. BLE‑пульт блокировки
Отдельный BLE‑сервис для удалённой блокировки/разблокировки (например, с телефона или брелока‑компаньона).

| Параметр | Значение |
|---|---|
| Имя устройства | `VESC-Lock` |
| Сервис | `a1b2c3d4-0001-4a5b-8c6d-1234567890ab` |
| Характеристика команд | `...-0002-...` (write) |
| Характеристика статуса | `...-0003-...` (read/notify) |

### 4. OTA — обновление «по воздуху»
Подключение к домашней Wi‑Fi сети (вводится на экране Wi‑Fi) и прошивка через `ArduinoOTA`. Таймаут подключения — 15 с.

> ℹ️ Wi‑Fi AP (дашборд), BLE‑мост и OTA — взаимоисключающие режимы: при включении одного остальные корректно останавливаются, чтобы хватало памяти.

## 🗂 Структура экранов

```
HOME ─┬─ STATS ─── (рекорды/статистика)
      ├─ SETTINGS ──┬─ Lock / PIN
      │             ├─ BT Bridge (VESC Tool)
      │             ├─ Diag
      │             ├─ Motor config
      │             ├─ Records (+ reset)
      │             ├─ Wi‑Fi / OTA
      │             ├─ SETTINGS 2 ─┬─ Cells
      │             │              ├─ Web Dash
      │             │              └─ Wi‑Fi config (keyboard)
      │             └─ SETTINGS 3 ─┬─ Dragy
      │                            ├─ Auto‑lock
      │                            ├─ Speed limit
      │                            ├─ Music
      │                            └─ CAN ID
      └─ (свайп/тач навигация)
```
